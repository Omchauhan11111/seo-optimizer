Phase 0 — Content Loading & Extraction
How DOCX and URL content is parsed

DOCX Mode (mammoth.js library):

  • The uploaded .docx file is read as an ArrayBuffer by FileReader
  • Mammoth converts it to HTML using custom style maps (heading styles → H1/H2/H3, custom paragraph styles → FAQ Q/A)
  • A structured text extraction function walks the HTML DOM, adding H1:, H2:, FAQ Q:, FAQ A: prefixes for AI context
  • Tables are extracted as pipe-separated rows: | Cell 1 | Cell 2 |
  • Limitation: WordPress accordion FAQ answers and TOC plugins are JavaScript-rendered — they don't exist in the DOCX file at all

URL Fetch Mode (recommended for WordPress pages):

  • The live URL is fetched through a CORS proxy (allorigins.win → corsproxy.io → cors-anywhere, tried in order)
  • The full rendered HTML is parsed by DOMParser — this includes all accordion content that is in the DOM even if hidden via CSS
  • Hidden divs (class="hidden", style="display:none") are explicitly made visible before text extraction
  • TOC links (lwptoc plugin, Easy TOC, etc.) are extracted and prepended as a structured list
  • Accordion questions and answers are detected by class names: accordion_title, accordion_content, faq-q, faq-a, common_content_style
  • Navigation, headers, footers, sidebars, and widgets are stripped to keep only page body content
✅ URL fetch extracts exactly what Google's crawler sees — the actual text content of your live page including all dynamic elements.

Market & Brand Detection
How the AI knows which country/company you're optimizing for

The detectMarket() function runs before every AI call. It determines the target country in this priority order:

  1. Page URL domain extension (most reliable): .my → Malaysia, .sg → Singapore, .com.au → Australia, .in → India, .hk → Hong Kong
  2. URL path/subdomain keywords: /singapore/, /malaysia/ in the URL
  3. Document text signal counting: counts occurrences of country-specific terms (e.g. "Ringgit/RM/SSM/Sdn Bhd" = Malaysia; "ACRA/CPF/SGD/Pte Ltd" = Singapore). Needs 2+ signals to trigger
  4. Topic text: if topic contains "singapore" or "malaysia"
  5. Fallback: uses "the target market" (generic) — this triggers when no URL is provided

The detectBrand() function extracts the company name from the domain: 3ecpa.com.my → 3ecpa, then capitalizes it.

⚠️ Always enter the Page URL in Step 4. Without it, market detection falls to document text scanning which may be unreliable. The URL domain is the fastest and most accurate signal.

The 4 AI Calls — What Each One Does
Sequential API calls: each feeds context into the next

The tool makes 4 sequential API calls. Each call's output is fed as context into the next, building a progressively richer picture of the content.

Call 1 — Keyword Research (pKw())

  • Sends: document preview (first 3,000 chars), topic, category, page type, market, page URL
  • Returns: 14 keywords as JSON → 3 primary, 5 secondary, 6 LSI with volume/difficulty/intent
  • How market affects output: adds country modifier to keywords (e.g. "HR outsourcing Malaysia")
  • How doc affects output: keywords are derived from actual content found in the document, not generated generically

Call 2 — SERP Analysis (pSERP())

  • Sends: document preview (2,000 chars), topic, market, page URL, page type
  • Returns: 5 competitor pages with URLs, word counts, sections, FAQ/table flags, content gaps, word count target, recommended sections
  • The AI generates realistic competitor data for the correct country — Malaysian competitors for .my sites, Singaporean for .sg sites
  • word_count_target: 850 for service pages, 2,500 for blog/guides

Call 3 — SEO Audit (pAudit())

  • Sends: full document (up to 4,500 chars), topic, market, word count, URL
  • Returns: score_before (honest current state), score_after (projected), breakdown by 6 dimensions, issues_fail/warn/pass
  • Score calculation: based on actual word count and content quality observed, not inflated
  • Issues are specific to the actual text — references real problems found in the document

Call 4 — Content Optimization (pOpt())

  • Sends: full document (5,500 chars), all 3 previous results, all checked option rules, market, brand, site URL, year, additional instructions
  • Returns: meta_title, meta_description, url_slug, h1, optimized_html (full page), changes_added, changes_removed
  • Content minimums enforced: 800w for service pages, 2,200w for blog/guides
  • Internal links use extracted site base URL: https://www.3ecpa.com.my/
ℹ️ Why sequential? The optimization call (Call 4) needs the keyword list and SERP gaps to know what to add, and the audit to know what's broken. Feeding all three as context produces far better output than one combined prompt.

Color-Coding System — How Changes Are Marked
Green = add, Red = delete, White = keep

The AI is instructed to wrap its output in specific HTML classes. CSS in styles.css applies visual styling:

HTML ClassDisplayMeaningAction for Editor
div.new-blockGreen backgroundEntire new section addedADD this section to your page
div.remove-blockRed + strikethroughExisting section to deleteREMOVE this from your page
span.new-inlineGreen highlightWord/phrase added inlineAdd this word/phrase in context
span.remove-inlineRed strikethrough inlineWord/phrase to delete inlineDelete this word/phrase
No wrapperNormal whiteKeep exactly as-isDO NOT change this content

How to implement changes in WordPress:

  1. Open the optimized content section in the report
  2. Read through from top to bottom
  3. For each GREEN section: copy the content and paste it into your WordPress editor at the indicated position
  4. For each RED section: find and delete the equivalent text from your WordPress page
  5. White/normal content: leave unchanged in WordPress
  6. Update meta title and description in your SEO plugin (Yoast, Rank Math, etc.)
✅ Best practice: Download the HTML report and open it alongside your WordPress editor. Work through the changes section by section.

Editable Prompt System
How to customize AI instructions for specific pages

In Step 5, clicking "👁 Preview & Edit AI Prompts" opens a panel showing the 3 analysis prompts before they run. This is useful when:

  • You want to add industry-specific context the AI might not know
  • You want to change the competitor sites for SERP analysis
  • You need to adjust scoring criteria for a specific content type
  • A prompt keeps producing wrong output for a particular page type

How it works:

  • Prompts are built fresh from the current document, topic, market, and URL every time you click the button
  • Clicking "Edit Prompt" on any prompt opens a full-screen editor with the live-generated text
  • Your edits are saved in window.promptOverrides for the current session only
  • Edited prompts show an [EDITED] badge so you know which ones are customized
  • "↩ Reset" restores the auto-generated default
  • The optimization prompt (Call 4) is always auto-generated — it combines the outputs of Calls 1-3, so editing it beforehand isn't possible

Score Interpretation
What the numbers mean
Score RangeMeaningTypical Cause
10–25Very poor — needs complete rewriteThin content (<300w), no headings, no keywords
25–45Poor — significant gapsShort content, missing structure, no CTAs
45–65Average — exists but underoptimizedDecent content but lacks keywords, freshness, E-E-A-T
65–80Good — minor improvements neededSolid content, some keyword/structure gaps
80–92Excellent — well optimizedFull optimization applied

Score breakdown dimensions:

  • Content Depth — word count, comprehensiveness, sub-topic coverage
  • Keyword Usage — primary/secondary keywords in headings, density, LSI usage
  • Content Freshness — current year data, updated fees/regulations/stats
  • Structure / UX — heading hierarchy, TOC, bullet lists, readability
  • E-E-A-T Signals — expertise/authority indicators, citations, credentials
  • CTA / Conversion — call-to-action presence, internal links, contact pathways
